# GitHub Discussions Guide

Use Discussions to make aihelper easier to adopt and easier to trust.

## Recommended Categories

| Category | Purpose |
|---|---|
| Announcements | Releases, benchmark posts, launch updates |
| Q&A | Install help, editor setup, local model questions |
| Show and tell | User workflows, demo GIFs, benchmark screenshots |
| Ideas | Feature proposals framed around concrete workflow pain |
| Benchmarks | Latency, token reduction, and local-first runtime results |

## First Pinned Discussion

Title:

```text
What workflow should aihelper optimize next?
```

Body:

```markdown
aihelper exists to replace giant repo scans with semantic routing, compact context,
and patch-first editing.

What workflow should be easier?

- diagnostics -> patch
- semantic routing for large repos
- editor MCP setup
- OCR/screenshot parsing
- benchmark or telemetry visibility

Please include the command, editor, repo size, and where the current workflow feels slow.
```

## Moderation Notes

- Ask for `aihelper doctor` output when debugging setup issues.
- Move broad ideas toward concrete workflows.
- Prefer benchmark screenshots, repro commands, and small examples over abstract requests.
