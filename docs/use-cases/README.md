# Use Cases

## Purpose

Use-case docs explain the business flow in plain language and then map that flow to technical behavior.

## Each Use Case Should Include

- Purpose
- Entry points
- Main steps
- Data touched
- Service calls
- Integrations
- `ext` overrides or local overrides
- Related use cases

## Template

```text
# <Use Case Name>

## Purpose

## Entry Points

## Steps

## Data

## Services

## Integrations

## Overrides

## Notes
```
