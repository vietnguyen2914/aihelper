## 2. Workflow Composition: Before vs After

### Before (v0.0.8 — initial v0.0.9 design)

Each workflow duplicated handler logic. Example: `diagnose` workflow had 6 phases, each calling a single handler:

```yaml
phases:
  - name: analyze_target      # handler: analyze_target
  - name: retrieve_memory     # handler: retrieve_memory  
  - name: trace_callers       # handler: trace_callers
  - name: trace_callees       # handler: trace_callees
  - name: hypothesize_fix     # kind: local_model
  - name: verify_fix          # handler: run_tests
```

**Problem:** `analyze_target`, `retrieve_memory`, `trace_callers` duplicated across 3-4 workflows.

### After (v0.0.9 with primitives)

Single composed phase using `uses:` syntax:

```yaml
phases:
  - name: analyze
    uses: [graph.analyze_target, memory.recall, graph.trace_callers, graph.trace_callees]
  - name: hypothesize
    kind: local_model
    model: qwen3.5:4b-16k
  - name: verify
    uses: [test.run]
```

**Result:** 6 phases → 3 phases. Zero handler duplication. Primitives are shared across all workflows.

## 3. Observability: New in v0.0.9

Every workflow run now records:

```json
{
  "trace_id": "a3f8b2c1",
  "timestamp": "2026-05-28T14:30:00Z",
  "workflow": "release_check",
  "success": true,
  "total_tokens": 450,
  "total_duration_ms": 1205.3,
  "ai_calls": 1,
  "deterministic_steps": 1,
  "token_breakdown": {
    "deterministic": 0,
    "local_model": 450,
    "frontier": 0
  }
}
```

This enables querying: "which workflow is most expensive?", "what's the average token usage?", "how often does tier router misclassify?"

## 9. Conclusion

v0.0.9 with primitives composition achieves:

1. **Zero handler duplication** — 17 primitives compose into 5 workflows
2. **Observability** — Every run traced with trace_id, timestamps, token breakdown
3. **Compression safety** — 6 fidelity checks prevent context loss
4. **Maintainability** — New workflow = ~12 lines of YAML composing existing primitives
5. **Token cost** — $0.54/month (was $8.88 in v0.0.8, $19.40 in v0.0.7)

The architecture is now a **composable, observable, deterministic-first engineering cognition runtime** — not an AI wrapper.
