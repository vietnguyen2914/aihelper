# aihelper v0.0.9 — Workflow Runtime & Strategic Cognition

**Released:** 2026-05-28

> **"Skills should compile into executable workflows. Not prompts."**

v0.0.9 introduces the Workflow Runtime Engine — the missing layer that transforms
aihelper from a context orchestrator into a **Local Engineering Cognition Runtime**.

---

## 🚀 What's New

### 1. Workflow Runtime Engine (`workflow_engine.py`)

A lightweight state machine that executes deterministic engineering workflows
with **zero AI token usage**. Workflows are defined declaratively and executed
by the daemon, calling AI only at decision points.

```bash
# New CLI commands
aihelper workflow run tdd --target src/UserService.ts
aihelper workflow run diagnose --error "TypeError in auth middleware"
aihelper workflow run release-check
aihelper workflow run architecture-review --scope auth-module
aihelper workflow list
```

**Built-in workflows:**

| Workflow | Steps | AI Calls |
|---|---|---|
| `tdd` | analyze → stub → fail → implement → pass → regression → summary | 2 (test generation, implementation) |
| `diagnose` | reproduce → trace → inspect → hypothesize → fix → verify | 2 (hypothesis, fix) |
| `release-check` | tests → lint → architecture → dependencies → git-diff → summary | 1 (summary) |
| `architecture-review` | graph → callers → impact → memory → constraints → report | 1 (report) |
| `refactor-safety` | trace → callers → impact → risk → strategy | 1 (strategy) |

### 2. Tier Router (`tier_router.py`)

Explicit routing logic that classifies every task into the correct execution tier:

```python
route = tier_router.classify(task)
# → {"tier": "deterministic", "handler": "workflow_engine.run"}
# → {"tier": "local_model", "handler": "ollama.run", "model": "qwen3.5:4b-16k"}
# → {"tier": "frontier", "handler": "escalate", "reason": "high_ambiguity"}
```

**Ambiguity scoring:** 0.0–1.0 based on:
- Task novelty (seen before?)
- Constraint conflicts
- Historical failure rate
- Graph complexity (callers/callees depth)
- Confidence of sub-decisions

### 3. Verification Runtime (`verify.py`)

Reusable verification commands wrapping the existing graph + validation primitives:

```bash
aihelper verify architecture         # Check architectural rules
aihelper verify auth-safety          # Audit auth flow security
aihelper verify regression-risk      # Predict regression impact
aihelper verify dependency-health    # Check circular deps, dead code
```

### 4. Context Compressor (`compressor.py`) + Compression Fidelity (`compressor_fidelity.py`)

Builds distilled cognition packages for frontier models with **incremental caching**.
Same target + unchanged graph = cache hit, no recomputation.

```json
{
  "system_state": {
    "architecture": {...},
    "historical_failures": [...],
    "current_change": {...},
    "affected_graph": [...],
    "risks": [...]
  },
  "question": "..."
}
```

**Compression ratios + fidelity:** 6 automated checks ensure context preservation.

### 5. Primitives Registry + Execution DAG (`primitives.py`)

17 reusable primitives with `PrimitiveContract` declaring inputs, outputs, cacheability,
dependencies, and cost estimates. `build_execution_dag()` groups primitives into parallel
stages based on dependency analysis. Cache hits skip execution entirely.

### 6. Incremental Architecture (v0.0.7 substrate + v0.0.9 contracts)

aihelper converges with compiler architecture — not agent frameworks:

| Concept | Version |
|---|---|
| File invalidation (`cache_diff`) | v0.0.7 |

## 🎯 Design Decisions

1. **Workflows are YAML, not markdown** — Declarative, machine-executable, no LLM parsing needed
2. **State machine, not prompt chain** — Phases with evidence gates, retry, rollback
3. **Tier routing is explicit** — No ambiguity about whether AI is needed
4. **Compression is lossless for engineering facts** — Only removes what Python can deterministically resolve
5. **Zero new dependencies** — Python stdlib (`yaml` already in requirements)

### Design Principles
- **Deterministic-first** — Execute locally whenever possible
- **AI-at-decision-points** — Only call LLMs when ambiguity threshold exceeded
- **State machine reliability** — Evidence gates, retry, rollback
- **Zero new dependencies** — All Python stdlib
- **Compression over context** — Send distilled knowledge, not raw files
