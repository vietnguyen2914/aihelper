## 🚀 What's New in v0.0.7

### 1. SQLite + FTS5 Knowledge Graph (P0 Foundation)

The biggest architectural upgrade: aihelper now has a **SQLite-backed knowledge graph with FTS5 full-text search**, running alongside the existing JSON cache.

- **FTS5 full-text search**: **0.032ms/query** — 15x faster than JSON iteration (0.5ms)
- **WAL mode**: concurrent reads never block the writer
- **Indexed queries**: `ORDER BY`, `LIMIT`, `WHERE kind/file` at SQL speed
- **Atomic transactions**: no corrupt caches on crash
- **Zero dependencies**: Python built-in `sqlite3` module
- **Automatic sync**: `aihelper cache build` now syncs to SQLite transparently — all your `~/.ai-cache/*.json` files remain untouched and functional

```bash
$ aihelper graph status
# symbol_count: 650  |  edge_count: 424  |  file_count: 102
# db_size: 0.64MB  |  journal: wal
# nodes_by_kind: {class: 36, function: 393, method: 3, const: 5, file: 213}
```

### 2. Six New MCP Tools: Call Graph Queries

**Total MCP tools: 15 (was 9).**

| Tool | Description | Savings vs Agent-native |
|---|---|---|
| `aihelper_callers` | All symbols calling a function (BFS, depth-configurable) | 5-10 grep/Read calls → **1 call** |
| `aihelper_callees` | All symbols called by a function | 5-10 grep/Read calls → **1 call** |
| `aihelper_trace` | BFS shortest call path between two symbols | g**rep structurally cannot do this** → **1 call** |
| `aihelper_impact` | Transitive impact radius (callers + importers) | Full repo scan → **1 call** |
| `aihelper_explore` | Multi-symbol source in ONE call (grouped by file, line-numbered) | 4-8 Read calls → **1 call** |
| `aihelper_graph_status` | Knowledge graph health and statistics | Filesystem scan → **instant** |

All 6 new tools proxy through the **persistent daemon (0.3ms IPC)** — zero cold startup cost.

### 3. New CLI: `aihelper graph`

```bash
aihelper graph callers --symbol UserService
aihelper graph callees --symbol getUser --depth 2
aihelper graph trace --from QuerySet --to execute_sql
aihelper graph impact --symbol processPayment --depth 3
aihelper graph explore --query "AuthService loginUser session"
aihelper graph status
```

### 4. `aihelper upgrade` — Auto-Migration for All Projects

One command scans `~/github/*`, rebuilds all caches with SQLite sync:

```bash
aihelper upgrade
# ✅ aihelper (650 symbols, 424 edges, 0.64MB SQLite)
# ✅ project1 (47231 symbols, 12840 edges, 4.2MB SQLite)
# ✅ project2 (3812 symbols, 903 edges, 0.9MB SQLite)
```

Each project's existing JSON files are preserved — SQLite is additive.

### 5. `aihelper affected` — Test-File Impact from Git Diff

Given changed source files, finds test files that depend on them via the SQLite import graph:

```bash
aihelper affected src/utils.py src/api.py
git diff --name-only | aihelper affected --stdin
aihelper affected src/auth.ts --filter "e2e/*"

# Output:
# ## Affected Tests
# Changed: src/cache.py
# Affected tests: 3
#   - tests/test_cache.py
#   - tests/test_persistence.py
#   - tests/integration/test_sync.py
```

Configurable: `--depth` (dependency traversal, default 5), `--filter` (custom test-file glob), `--stdin` (pipe from git).

### 6. Tree-sitter AST Extraction — Now Default

`requirements.txt` now includes `tree-sitter` + 9 language grammars. When installed, `ast_extractor.py` provides **precise AST-level extraction** with call edges, class hierarchy, and interface/struct support. Falls back to regex extraction otherwise.

```bash
pip install -r requirements.txt   # One command, includes tree-sitter
```

**Supported languages:** Python, JavaScript, TypeScript, Java, Go, Rust, PHP, C, C++.

**Call edges vs regex:** tree-sitter creates exact `calls` edges between symbols. Regex extraction creates file-level `imports` edges. With `pip install`, you get AST precision automatically.

### 7. Framework-Aware Route Detection

`framework_routes.py` detects web routes in **14 frameworks** and links them to handler symbols in the knowledge graph:

| Framework | Detection pattern |
|---|---|
| Django | `path()`, `re_path()`, `include()` in `urls.py` |
| Flask | `@app.route('/path', methods=[...])` |
| FastAPI | `@app.get()`, `@router.post()`, `@app.put()` |
| Express | `app.get()`, `router.post()` |
| NestJS | `@Controller` + `@Get/@Post/...` |
| Laravel | `Route::get()`, `Route::resource()` |
| Spring | `@GetMapping`, `@RequestMapping` |
| Gin / chi / gorilla | `r.GET()`, `r.Handle()` |
| Axum / actix / Rocket | `.route("/", get(handler))` |
| ASP.NET | `[HttpGet("/path")]` attributes |
| React Router | `<Route path="/" component={X} />` |

### 8. Native File Watcher (watchdog)

`file_watcher.py` provides native OS file watching via watchdog (FSEvents/inotify/ReadDirectoryChangesW) with 2-second debounce, plus polling fallback. This upgrades and coexists with the existing Watchman integration.

## 📊 Benchmarks

### aihelper Self-Host (M1 Pro, 32GB, 102 files)

Methodology: 4 runs per metric, median reported (matching codegraph's methodology).

| Metric | Median | vs JSON baseline |
|---|---|---|
| **Cache build** (cold) | **12ms** | n/a |
| **FTS5 search** (per query) | **0.032ms** | **15x faster** (JSON: 0.5ms) |
| **JSON lookup** (per query, baseline) | **0.5ms** | n/a |
| **Call graph query** (callers/callees) | **0.8ms** | **not available before v0.0.7** |
| **SQLite DB size** (102 files, 650 symbols) | **0.64MB** | JSON: 0.45MB + DB = 1.1MB total |

**Key insight:** FTS5 is **0.032ms/query** — you can run 30 searches in 1ms. The JSON baseline is 0.5ms because it iterates the entire symbol list in Python. SQLite's indexed B-tree lookup is O(log n).

### aihelper v0.0.6 → v0.0.7 Performance Gains

| Operation | v0.0.6 | v0.0.7 | Gain |
|---|---|---|---|
| Symbol lookup | 0.5ms (full JSON iteration) | **0.032ms** (FTS5 indexed) | **15x** |
| Callers query | **not available** (agent grep/Read loop) | **0.8ms** (SQLite recursive CTE) | **enabled** |
| Trace query | **not available** (agent sub-agent exploration) | **1.2ms** (BFS on edges) | **enabled** |
| Impact analysis | **heuristic** (line-regex scanning) | **1.5ms** (transitive closure via SQLite) | **exact** |
| Multi-symbol explore | N Read calls (agent-driven) | **1 call** `aihelper_explore` | **80% fewer calls** |
| File watching | Watchman only | watchdog + Watchman | **redundant** |

## 📝 Full Changelog

### Added (8 new modules)
- `graph_db.py` (712 lines) — SQLite + FTS5 knowledge graph. Symbols, edges, files, recursive CTE BFS traversal, dead code detection, circular dependency detection, framework route storage.
- `ast_extractor.py` (593 lines) — Tree-sitter AST extraction. **9 languages** with S-expression queries. `ExtractionOrchestrator` pattern (tree-sitter → fallback regex).
- `framework_routes.py` (306 lines) — Web framework route detection. **14 frameworks** (Django, Flask, FastAPI, Express, NestJS, Laravel, Spring, Gin, Axum, actix, ASP.NET, React Router).
- `file_watcher.py` (210 lines) — Native OS file watcher via watchdog. 2-second debounce, polling fallback. `FileWatcher` unified interface.
- `graph_query.py` (301 lines) — MCP tool handlers for callers, callees, trace, impact, explore. Uses SQLite FTS5 with regex fallback.
- `affected.py` (151 lines) — Test-file impact analysis from git diff. Transitive import dependency traversal via SQLite.
- `scripts/benchmark.py` (140 lines) — Benchmark suite. 4 runs per metric, median reported. Measures cache build, FTS5, symbol lookup, call graph.
- `docs/releases/v0.0.7.md` (this file)

### Changed (7 existing modules)
- `cache.py` — `_sync_cache_to_sqlite()` auto-syncs JSON caches to SQLite on every `build_cache`. `_detect_language()` helper. `sqlite_synced` in manifest. **New incremental path:** `update_cache()`, `build_file_index_incremental()`, `build_symbol_graph_incremental()`, `sync_sqlite_incremental()`. `clean_cache()` now resets SQLite singleton via `close_all()`. `watch_cache()` uses `update_cache()` instead of `warm_project()`.
- `mcp_server.py` — 15 MCP tools (was 9). Added `_callers_tool_schema`, `_callees_tool_schema`, `_trace_tool_schema`, `_impact_tool_schema`, `_explore_tool_schema`, `_graph_status_tool_schema`. `_call_graph_tool()` helper. All graph tools proxy through daemon.
- `daemon.py` — 6 new handler methods: `handle_graph_callers`, `handle_graph_callees`, `handle_graph_trace`, `handle_graph_impact`, `handle_graph_explore`, `handle_graph_status`. 49 total dispatchable methods.
- `main.py` — Added `upgrade`, `graph`, `affected` subcommands. `_handle_upgrade()`, `_handle_graph()`, `_handle_affected_cli()` handler functions.
- `requirements.txt` — tree-sitter (9 language grammars) + watchdog now default dependencies.

### Fixed
- **JSON file storage O(n) bottleneck** → SQLite FTS5 O(log n). Symbol lookup went from 0.5ms to **0.032ms**.
- **No call graph** → Multi-depth BFS callers, callees, trace, impact via SQLite recursive CTEs.
- **No framework routes** → 14 web frameworks detected with edges from routes → handlers.
- **`clean_cache` not resetting SQLite singleton** → now calls `close_all()` to prevent stale DB connections after clean.
- **`watch_cache` always doing full rebuild on every edit** → replaced by `update_cache()` incremental path. `cache_diff()` detects file changes → `build_file_index_incremental()` + `build_symbol_graph_incremental()` + `sync_sqlite_incremental()` process **only changed files**. No more scanning all 4000 files on every save.

## 🔧 Installation & Upgrade

### Fresh install
```bash
git clone https://github.com/vietnguyen2914/aihelper.git
cd aihelper
python3 -m pip install -r requirements.txt  # includes tree-sitter + watchdog
./bin/aihelper doctor
```

### Upgrade from v0.0.6
```bash
cd aihelper
git pull
python3 -m pip install -r requirements.txt  # gets tree-sitter + watchdog
aihelper upgrade                              # one command, migrates all projects
```

### Run benchmarks on your machine
```bash
python3 scripts/benchmark.py                              # all ~/github projects
python3 scripts/benchmark.py --project-root /path/to/project  # single project
python3 scripts/benchmark.py --json                        # JSON output
```

### Verify everything works
```bash
aihelper graph status
aihelper graph callers --symbol main
aihelper affected --stdin <<< "$(git diff --name-only HEAD)"
```
