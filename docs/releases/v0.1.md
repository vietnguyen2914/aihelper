# aihelper v0.1 — Kernel Hardening: Typed Execution & Semantic Invalidation

**Released:** 2026-05-28

> **"Harden the kernel. Don't expand the feature surface."**

v0.1 does not add a single new feature. Instead, it **wires together** the
compiler-inspired modules that v0.0.9 introduced in isolation, transforming
them from standalone components into a unified **Incremental Cognition Compiler**.

---

## 🎯 The Wiring Gap (v0.0.9 → v0.1)

v0.0.9 introduced 10 brilliant modules but left them **architecturally isolated**:

| Module | v0.0.9 Status | v0.1 Status |
|---|---|---|
| `optimizer.py` | Written, never called from `workflow_engine.py` | ✅ Wired into execution pipeline |
| `invalidation.py` | Daemon logging only, never called from `cache.py` | ✅ Wired into cache update cycle |
| `compressor.py` | No confidence/decay tracking | ✅ Weighted confidence decay |
| `PrimitiveContract` | Basic fields (cacheable, cost_estimate) | ✅ Typed capabilities (purity, determinism, scope, parallel_safe) |

## 🚀 What Changed

### 1. Typed Execution Capabilities (`primitives.py`)

`PrimitiveContract` now carries immutable capability metadata that the optimizer
uses to make safe parallelization and caching decisions:

```python
PrimitiveContract(
    purity="pure",              # "pure" | "mutative" | "unknown"
    determinism="deterministic", # "deterministic" | "heuristic" | "ai_bound"
    invalidation_scope="symbol", # "symbol" | "file" | "module" | "global"
    parallel_safe=True,          # Can run concurrently
)
```

**Smart defaults** inferred from existing fields:
- `side_effects=True` → `purity="mutative"`, `parallel_safe=False`
- `side_effects=False` → `purity="pure"`, `parallel_safe=True`

**Distribution on aihelper self-host (17 primitives):**
- 14 pure + 3 mutative
- 14 parallel-safe + 3 sequential
- All use `symbol` invalidation scope

### 2. Optimizer Wiring (`optimizer.py` → `workflow_engine.py`)

The optimizer now runs **before** DAG staging in `_execute_primitives()`:

```
workflow YAML → primitive list → optimize_dag() → build_execution_dag() → execute
```

`optimize_dag()` returns `OptimizationResult` with full profiling:
- `applied_passes`: which optimizations ran
- `folded_nodes` / `cache_hits`: what was constant-folded
- `eliminated_nodes`: what was deduplicated
- `estimated_speedup`: cost-based speedup estimate

**Optimizer remains PURE** — no filesystem, graph, or cache mutation.

### 3. Semantic Invalidation Wiring (`invalidation.py` → `cache.py`)

Every cache update now classifies file changes with `classify_change()`:

| Change Type | Propagation | Semantic Confidence |
|---|---|---|
| Signature changed, high confidence | ✅ Targeted (symbol-level) | > 0.80 |
| Signature changed, low confidence | ⚠️ Conservative (file-level) | ≤ 0.80 |
| Body-only, high confidence | ❌ Skip | ≥ 0.75 |
| Body-only, low confidence | ⚠️ Conservative | < 0.65 |
| High-risk module (auth/security/payment) | ⚠️ Always propagate | — |

New daemon handler: `invalidation_classify` — exposes semantic invalidation to MCP/CLI.

### 4. Compression Confidence Decay (`compressor.py` + `invalidation.py`)

Weighted decay replaces fixed thresholds:

| Change Type | Decay | Meaning |
|---|---|---|
| `body_only_change` | 0.01 | Negligible impact — 100 changes to recompress |
| `signature_change` | 0.08 | Moderate — ~5 signature changes to recompress |
| `architectural_hotspot` | 0.15 | High — ~3 structural changes to recompress |
| `branch_switch` | 0.40 | Full shift — triggers recompression immediately |
| `large_churn` | 0.25 | Many files changed |
| `dependency_change` | 0.10 | Import/dependency changed |

**Recompression threshold:** 0.60 — below this, compression cache is cleared and rebuilt.

**Conservative fallback:** High-risk modules (auth, security, payment) get 1.5× decay boost.

## 🎯 Design Decisions

1. **Typed capabilities are immutable** — set at registration, never mutated at runtime. This prevents optimizer behavior instability.

2. **Optimizer operates on IR** — input is `List[str]`, output is `OptimizationResult`. No YAML, no workflow dependency. Clean separation.

3. **Weighted decay over fixed thresholds** — eval2's refinement. More stable than arbitrary constants, adapts to change type severity.

4. **Conservative fallback for high-risk modules** — auth, security, payment always propagate invalidation and get elevated decay. Safety over optimization.

5. **Zero new dependencies** — all Python stdlib. No external packages added.

6. **Backward compatible** — all existing primitives, workflows, and MCP tools unchanged. New fields have safe defaults.

## 📝 Full Changelog

### Added
- `PrimitiveContract`: 4 typed capability fields (`purity`, `determinism`, `invalidation_scope`, `parallel_safe`)
- `OptimizationResult`: profiling dataclass with `applied_passes`, `folded_nodes`, `cache_hits`, `eliminated_nodes`, `estimated_speedup`
- `ChangeClassification`: semantic confidence-scored change classification
- `WEIGHTED_DECAY_TABLE`: 7 change-type-specific decay rates
- `HIGH_RISK_MODULE_PATTERNS`: auth/security/payment module detection
- `should_propagate_invalidation()`: call-graph-aware invalidation decisions
- `compute_semantic_confidence()`: confidence scoring for change classification
- `compute_compression_confidence()`: weighted decay application
- `apply_compression_decay()`: compressor-level decay tracking
- `_apply_semantic_invalidation()`: cache-level semantic invalidation
- `handle_invalidation_classify`: new daemon handler

### Changed
- `optimizer.py`: `optimize_dag()` now returns `OptimizationResult` (was `List[str]`)
- `optimizer.py`: `constant_folding_pass()` now checks `is_pure` before caching
- `optimizer.py`: Fixed dedup eliminated_nodes tracking (was broken for set-based counting)
- `workflow_engine.py`: `_execute_primitives()` calls optimizer before DAG staging
- `workflow_engine.py`: Cache-hit skipping integrated with optimizer results
- `cache.py`: `update_cache()` runs semantic invalidation on changed files
- `cache.py`: `update_cache()` returns `invalidation_report` in result
- `daemon.py`: +1 handler (`invalidation_classify`), 55 total (was 54)
- `primitives.py`: `PrimitiveContract.to_dict()` includes typed capability fields

### Design Principles
- **Harden the kernel** — No new features, pure integration
- **Immutable capabilities** — Set at registration, never mutated
- **Pure optimizer** — No filesystem, graph, or cache mutation
- **Conservative invalidation** — High-risk modules always propagate
- **Weighted decay** — Proportional to change severity
- **Zero new dependencies** — All Python stdlib
