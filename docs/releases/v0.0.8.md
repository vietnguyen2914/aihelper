# v0.0.8 — Cross-Platform Integration Overhaul

**Release date:** 2026-05-26

## Summary

Complete overhaul of aihelper's editor and agent integration layer. Added missing
integration scripts, created a shared Python core, documented all OS boundaries,
and ensured parity between macOS/Linux and Windows setups.

## New integration scripts

| Script | Target | What it does |
|--------|--------|-------------|
| `scripts/zed-integration.py` | Zed Editor | Writes `mcp_servers.aihelper` to `~/.config/zed/settings.json` |
| `scripts/gemini-integration.py` | Gemini / Antigravity | Writes `mcpServers.aihelper` to `~/.gemini/config/mcp_config.json` |
| `scripts/opencode-integration.py` | OpenCode | Writes `mcpServers.aihelper` to `~/.config/opencode/opencode.json` |

Previously, only VS Code/Copilot, Codex, and Claude had dedicated scripts.
Zed was handled by fragile inline bash code in `init-config.sh` that only
*patched* existing entries (never *created* new ones). Gemini and OpenCode
had no integration at all.

## Shared core: `scripts/integration_common.py`

All 6 integration scripts now import from a single shared module (~274 lines),
eliminating ~430 lines of duplicated code across individual scripts.

| Function | Previously duplicated across | Now lives in |
|----------|---------------------------|-------------|
| `load_json()` | codex, gemini, opencode, vscode-copilot, zed (5 copies) | `integration_common` |
| `write_json()` | codex, gemini, opencode, vscode-copilot, zed (5 copies) | `integration_common` |
| `merge_settings()` | gemini, opencode, vscode-copilot, zed (4 copies) | `integration_common` |
| `write_text()` | claude, vscode-copilot (2 copies) | `integration_common` |
| `ensure_extension_recommendation()` | vscode-copilot only (1 copy) | `integration_common` |
| `safe_run()` | vscode-copilot only (1 copy) | `integration_common` |
| `detect_binary()` | each script had custom `shutil.which()` calls | `integration_common` |
| `_script_dir()` / `_aihelper_root()` / `_mcp_server_path()` | gemini, opencode, zed (3 copies) | `integration_common` |
| `--dry-run` arg | all 6 scripts | `integration_common.add_dry_run_arg()` |
| `--path` arg | claude, vscode-copilot (2 copies) | `integration_common.add_path_arg()` |

## VS Code Copilot detection fix

VS Code 1.99+ bundles Copilot Chat as a built-in extension at
`/Applications/Visual Studio Code.app/Contents/Resources/app/extensions/copilot`.
The old detection code searched for `"github.copilot"` as a substring — but
`"github.copilot" in "copilot"` is always `False`. Fixed by adding bare
`"copilot"` to the match list and checking for exact directory name match.

## Windows parity fix

`scripts/init-config.ps1` (PowerShell, Windows) previously had its own inline
VS Code settings code and did **not** call any of the Python integration
scripts. Fixed:

- Added `Run-IntegrationScript` function (mirrors bash's `run_integration_script`)
- Added calls to all 6 Python integration scripts (same order as bash)
- Replaced inline VS Code settings code with `vscode-copilot-integration.py` call

## OS boundaries documented

Added a complete OS boundary audit table to `docs/integrations/editor-integration.md`:

| Script | Windows | macOS | Linux |
|--------|---------|-------|-------|
| `vscode-copilot-integration.py` | `%APPDATA%`, `%USERPROFILE%`, `%ProgramFiles%` | `~/Library/Application Support`, `/Applications` | `~/.config`, `/usr/share`, Flatpak |
| `zed-integration.py` | **N/A** (Zed not on Windows) | `~/Library/Application Support/Zed`, `/Applications/Zed.app` | `~/.config/zed` |
| `gemini-integration.py` | `%APPDATA%` | `~/.gemini` | `~/.gemini` |
| `opencode-integration.py` | `%APPDATA%` | `~/.config`, `~/Library/Application Support` | `~/.config` |
| `codex-integration.py` | `Path.home()` | `Path.home()` | `Path.home()` |
| `claude-integration.py` | `Path.home()` | `Path.home()` | `Path.home()` |

Platform-conditional code now uses single-source-of-truth constants
(`IS_WINDOWS`, `IS_MACOS`, `IS_LINUX`) from `integration_common` instead
of ad-hoc `sys.platform` checks scattered across files.

## AI instructions updated

Both `~/.github/copilot-instructions.md` (global, read by Copilot Chat) and
`aihelper/.github/copilot-instructions.md` (project-level) now include an
explicit **"Integration Scripts — DO NOT FORGET BOTH PLATFORMS"** section
that lists all 6 scripts and both init-config callers.

## Bash 3.2 compatibility fix

`init-config.sh` used `DRY_RUN_ARGS+=("--dry-run")` array syntax that triggered
an `unbound variable` error on macOS bash 3.2 when `--dry-run` was not passed.
Fixed by switching to a simple `DRY_FLAG` string variable.
