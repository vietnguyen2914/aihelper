# Project KB Generation Notes

## Goal

This repository is a reusable starter for building a project knowledge base before feature work begins.

## Delivery Order

1. Discover the project knowledge base.
2. Build the function mindmap in Mermaid.
3. Expand each function into use-case docs.
4. Add technical, data, integration, and override maps for each use case.
5. Build the function/use-case vs. system architecture matrix.
6. Generate `ai/index/*.json`, `ai/features/*.json`, and `ai/flows/*.json`.
7. Wire the docs together with clear links.

## Output Rule

- Keep the knowledge base compact.
- Prefer business meaning over framework/module names.
- Record `ext` or override paths whenever the project has them.
